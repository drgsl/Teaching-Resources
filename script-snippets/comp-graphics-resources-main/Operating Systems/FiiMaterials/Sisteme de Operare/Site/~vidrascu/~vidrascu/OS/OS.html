About the course

Objectives:
The students who will attend this course will obtain knowledge about operating systems, regarding the techniques used for the design and the implementation of them. Also, they will obtain the skills to use the UNIX/Linux operating system and to write parallel processing programs on Linux.

For additional details, please read the OS course's overview.

Curriculum

General thematics:

  • Basic concepts about operating systems. Architecture, components, services. System kernel.
  • Process management. Concurrency. Scheduling. Parallel processing. Process coordination. Interprocess communication.
  • Main memory management. Memory hierarchy. Allocation methods. Segmentation and paging. Virtual memory. Cache memory.
  • Secondary-storage management. File systems. Disk management.
  • Distributed systems. Distributed coordination. Types of distributed operating systems. Distributed file systems.

    Laboratories

    Laboratory thematics:

  • Part I: Overview of the UNIX/Linux operating system. Structure and general features. Guide of usage. Commands. Shells. Bash scripting language.
  • Part II: Concurrent programming in C language under Linux. Working with files. Exclusive/concurrent access to files. Process management. Creation, synchronization and executable loading. UNIX signals. Interprocess communication. Pipes. Fifos. Other communication mechanisms.

    Bibliography:

  • The book "Advanced Programming in the UNIX Environment" (3rd edition), by W. Richard Stevens, Stephen A. Rago, Publishing House Addison-Wesley Professional, 2013, ISBN 978-0321637734.

    Administratives:

    OS Laboratory consists of practice labs and some practical examinations, spanning over the all 13 weeks as follows:

    • Part I: weeks 1-5. Practice labs about: Overview of the UNIX/Linux operating system. Structure and general features. Guide of usage. Commands. Shells. Bash scripting language.
      Points for minimum presence and activity during these practice labs: 5 pts
      During some of these labs - one (or more) practical examination(s): 25 pts
    • Part II: weeks 6-13. Practice labs about: Concurrent programming in C language under Linux. Working with files. Exclusive/concurrent access to files. Process management. Creation, synchronization and executable loading. UNIX signals. Interprocess communication. Pipes. Fifos. Other communication mechanisms.
      Points for minimum presence and activity during these practice labs: 5 pts
      During some of these labs - one (or more) practical examination(s): 25 pts

    At the end of the semester, there will be also a theoretic examination (i.e. written test) : 40 pts

    Summarizing the above, the final mark is computed as follows:

    • Points for minimum presence and activity during the practice labs for both parts: 10 pts
    • 1st part practical examination(s): 25 pts
    • 2nd part practical examination(s): 25 pts
    • Final theoretic examination: 40 pts

    Minimum requirements for the passing of the course:

    • at least 20 points for all practical examinations
    • and at least 15 points at the final theoretic exam
