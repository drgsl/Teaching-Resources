Simulator grafic pentru algoritmii studiați la curs

Aplicație: OSCAL, i.e. Operating System Concept Animation Library

Este o aplicație formată din mai multe applet-uri Java, realizată în 2002 sub coordonarea profesorului Du Zhang, de la California State University, US.
Explicație: un applet Java este un program Java care "rulează" în cadrul unui browser de Internet, realizând de obicei o animație în cadrul paginii html în care este inclus.

Notă: aplicația are unele neajunsuri, în sensul că nu a mai fost "întreținută", iar tehnologia Java a evoluat mult de atunci.
Datorită riscurilor de securitate pe care tehnologia applet-urilor Java le presupunea, pe măsura trecerii timpului, principalele browsere de Internet existente au introdus restricții din ce în ce mai puternice referitor la rularea applet-urilor de Java în cadrul acestora.
Astfel, dacă în urmă cu mai mulți ani animațiile din OSCAL funcționaseră fără nicio problemă, mai apoi a fost nevoie de aprobarea explicită a utilizatorului pentru execuția unui applet în pagina pe care o viziona, iar în anii mai recenți se introduseseră mecanisme de tipul white-list prin care se configura browserul să accepte execuția de appleturi doar dacă proveneau dintr-o listă de adrese specificate explicit.
Iar începând din primăvara lui 2017, majoritatea browser-elor populare au renunțat complet la suportul pentru execuția de applet-uri Java! Singura soluție în acest caz, în afara aceleia de a folosi o versiune mai veche de browser ce nu are dezactivat suportul pentru plugin-ul de Java, este de a folosi comanda appletviewer la linia de comandă.

Modul de utilizare a aplicație OSCAL în prezent, ținând cont de restricțiile tehnologiei Java amintite mai sus:
- instalați un mediu Java (i.e., Oracle Java 8 SE) pe calculatorul local, ceea ce va instala și comanda appletviewer.
- descărcați de aici arhiva cu aplicația OSCAL și dezarhivați-o pe discul local, într-un director ales după dorință.
- pentru fiecare dintre exemplele enumerate mai jos, porniți animația de la linia de comandă, în directorul ales mai sus, introducând comanda: appletviewer calea-catre-applet , unde calea-catre-applet este calea, absolută sau relativă, către fișierul html ce conține applet-ul respectiv.

Puteți încerca acest simulator pentru a vă fixa sau pentru a aprofunda cunoștințele legate de subiectele abordate la curs.

  1. Animația #1: Simulator grafic pentru algoritmii de planificare a proceselor
    Funcționalitatea oferită: Algoritmii de planificare FCFS, SJF clasic (ne-preemtiv), SRTF (SJF preemptiv) și RR (cu cuanta=1 și cuanta=4).
    Calea relativă către applet: oscal/pscheduling.html
    Notă: deschideți și într-un browser acest fișier html, pentru a citi explicații suplimentare despre această animație.

  2. Animația #2: Simulator grafic pentru problema secțiunii critice
    Funcționalitatea oferită: Algoritmul Eisenberg-McGuire.
    Calea relativă către applet: oscal/EMFiles/EM.htm

    Calea relativă către applet: oscal/ProducerConsumer/ProducerConsumer.html
    ii) Funcționalitatea oferită: Problema cititori și scriitori.
    Calea relativă către applet: oscal/ReaderWriter/ReaderWriter.html
    iii) Funcționalitatea oferită: Problema bărbierului adormit.
    Calea relativă către applet: oscal/BarberShop/BarberShop.html
    iv) Funcționalitatea oferită: Problema producător-consumator rezolvată folosind semafoare.
    Calea relativă către applet: oscal/semaphore/semaphore.html
    Notă: deschideți și într-un browser acest fișier html, pentru a citi explicații suplimentare despre această animație.
    v) Funcționalitatea oferită: Problema producător-consumator rezolvată cu un monitor.
    Calea relativă către applet: oscal/monitor/monitor.html
    Notă: deschideți și într-un browser acest fișier html, pentru a citi explicații suplimentare despre această animație.
    vi) Funcționalitatea oferită: Mecanismul lock Read&Write din Solaris.
    Calea relativă către applet: oscal/SolarisRWLock/SolarisRWLock.html
    {Notă: Solaris-ul este un sistem de operare din familia UNIX, dezvoltat de firma SUN Microsystems (în prezent de către Oracle)}

    Funcționalitatea oferită: Algoritmii de planificare a discului FCFS, SSTF, SCAN, C-SCAN, LOOK și C-LOOK.
